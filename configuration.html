AWS Accounts

In order to load files from an S3 bucket you will need to provide AWS IAM API credentials. You provide these via the Preference dialog. In the AWS section of the Preferences click the + button in the displayed form enter:

  • Name : You can provide multiple sets of IAM credentials, this name is to identify the account
  • Account Number : The 12 digit AWS account number. This is optional though some functionality won't work if this is not provided.
  • Bucket Name : The name of the bucket where the CloudTrail files are located.
  • IAM Key : A IAM Key
  • IAM Secret : A IAM Secret

Note : The information entered here is stored in a database in your 'Home' directory on your computer so you only have to provide it once. Your credentials are not transmitted off your machine, don't believe me. read the code :)

Along with entering your information you will need to ensure that the IAM User has permissions to read the files from the S3 Bucket. A Policy like this should do the trick 

                {
                  "Version": "2012-10-17",
                  "Statement": [
                    {
                      "Effect": "Allow",
                      "Action": [
                        "s3:Get*",
                        "s3:List*"
                      ],
                      "Resource": "arn:aws:s3:::cloudtrail-log*",
                    }
                  ]
                }

AWS Alias

If you store the CloudTrail logs from multiple accounts in the same bucket remembering which account belongs to the account number can be a pain.

The AWS Alias section of the Preferences dialog allow you to put a friendly name to an account number. This friendly name will be displayed in the S3 FileChooser dialog.

Security

To configure what API Events are displayed on the Security Feature you use the Security section. Click the + button and select the AWS Service and API event that you want.

Resources

The Resources section allow you to define which Resource management API calls are displayed on the Resources Feature. Like the security section simply click the + button and select the API calls you are want to display

Scan Ignores

This section allows you to defined what API calls are ignored as part of a scan. Either type in a specific API calls such as ListClusters or a wild-carded value such as List*
