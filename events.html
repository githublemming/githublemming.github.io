Loading Events

Events can be loaded from two locations, the local file system or from an S3 bucket, and in two ways.

Full Load

Performing a Full Load of files will load every event in the file. The first two icons on the toolbar perform this action, the first icon is to load files from the local filesystem the second loads files from s3.

Loading complete files is useful when something has happened at a particular time and you want to see a flow of all activity across all services at that time. Loading complete files is memory intensive. It is not recommended that you load more that 100 files at the same time, though this is dependant on how busy your account is and how big the files are.

Scan

Scanning events is useful when you are only interesting in finding specific events, such as what actions has User x performed or who deleted my RDS!

  • The scan functionality for Local files is very basic, enter a string, then choose the local files
  • The scan functionality for S3 files is more advanced.

When you click the Scan S3 icon, that is the cloud with the magnifying class, a dialog will open that consists of two parts. One the left is the Filter Panel, on the right if the File Browser.

S3FileChooser

The S3FileChooser is a dialog that works similar to a standard FileChooser Dialog but is designed to display and allow select of files from an S3 bucket. From the top of the dialog it is possible to change between AWS accounts that you may have configured. If no AWS Accounts have been configured the dialog will not display and a warning advising you to register an AWS Account will be displayed.

The Dialog consists of two sections

  • The Filter Pannel
  • The FileChooser Panel

Filter Panel

The Filter Panel is located on the left side of the S3FileChooser dialog.

At the top of the Filter Panel is a dropdown control consisting of several filters and an Add button. The types of Filters that you can choose from include:

  • Text Filter : This allows you to search the whole CloudTrail event for a specific string.
  • Date Filter : This filter allow you to defined a from/to date that you are interested in searching from/to.
  • Ignore Filter : This filter will ignore all events that have been defined in the Ignore Scan section of preferences.
  • The other filters allow you to filter on a specific part of the CloudTrail event.

You can chooser how the filtering process works by selecting either AND or OR at the bottom of the panel.

FileChooser Panel

The filechooser panel works as a normal File Chooser, allowing you to select one or more files to be loaded. The load button will only enable when at least one filter has been defined correctly.

When performing a scan it is possible to select a folder and all files contained in that folder (and sub-folder) will be loaded and scanned.
