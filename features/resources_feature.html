Resources Feature

The Resources Feature shows user customisable Events.

The Resources Feature shows API calls that are to do with Creating, Modifying or Deleting of AWS Resources. Like the Security Feature what API calls this feature will handled is user configurable in the Preferences.


The Chart area of the side bar shows different information when using the Resources feature. Depending on the Event you select in the main section the chart options change. For example if you select 'RunInstances' you can see what AMIs have been used, or what EC2 Instances Types where started.

Clicking on a node will display a Table View containing just the Events for the node you select.

Configuration

While it could be said that every AWS Account is probably using EC2s, the same could not be said for the rest of the service. In order that the Resources Feature shows only the information that is relevant the Event that are shown on this section are customisable. Configuration is done via the Resources Section of the Preferences where you can add API events that you would like to view.

CloudTrail Viewer comes with some recommended API Events but you can remove these if you wish and replace them with your own. Simple use the + and - buttons at the bottom of the dialog.
